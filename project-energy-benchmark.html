Problem Statement
The Business Challenge

Energy-saving contracts (ESCs) require an accurate estimate of how much energy a building would have consumed without intervention — the counterfactual baseline. This number directly determines contract pricing and risk allocation. Without a reliable model, both sides are exposed to financial loss.

  • Buildings are highly heterogeneous — schools, restaurants, hotels each have fundamentally different consumption patterns
  • Initial predictions hovered around 70%, making stable pricing structurally impossible
  • Clients span multiple countries with inconsistent data formats and feature definitions
  • Goal: build a scalable counterfactual energy model that generalizes across diverse building types and data sources
Data Challenges
✕
Raw Data Problems
  • PDF and CSV formats mixed across clients
  • Inconsistent feature naming — same variable called differently per client
  • Missing and incomplete records throughout
  • Different feature sets available per building
  • No unified schema — each dataset effectively its own island
✓
How It Was Resolved
  • Standardized all inputs into a unified feature schema
  • Built a structured relational database across clients
  • Systematically identified missing fields and coordinated targeted data collection with clients
  • Remaining unavoidable gaps encoded as NA — handled natively by tree models
  • Feature importance analysis used to remove redundant and correlated variables

Modeling Approach
Algorithm
Tree-Based Ensemble

Random Forest and Gradient Boosted Trees — both natively handle missing values, non-linear feature interactions, and mixed-type inputs without requiring complete feature sets per sample.

Heterogeneity
One Model, All Buildings

Building type encoded as a categorical feature. Missing columns passed as NA — the model learns to route predictions appropriately. A single unified model replaces the need for per-client custom models.

Tuning
Feature Selection & Tuning

Feature importance analysis removed redundant and correlated variables. Hyperparameter tuning (tree depth, number of estimators, learning rate) optimized for cross-client generalization.
